# glossary.md 格式

`reference/glossary.md` 是这个教学工作区的标准用语，模板已**预置空骨架**。所有讲解、练习和学习记录都应当遵循其中的术语。**构建术语表本身就是学习的一部分**：能把一个概念压缩成一个紧致的定义，本身就是用户理解了它的证据。

## 结构

```md
# {主题} 术语表

{一两句话描述本术语表所覆盖的主题。}

## 术语

**Hypertrophy（肌肥大）**：
由机械张力与代谢压力在反复训练中驱动的肌肉生长。
**避免**：增肌、变大块头

**Progressive overload（渐进超负荷）**：
随着时间推移，系统性地增加施加在肌肉上的负担——通过负重、训练量或强度。
**避免**：再用力一点、升级

**RPE（自感用力等级，Rate of Perceived Exertion）**：
对一组训练有多吃力的 1–10 自评打分，10 表示力竭，8 表示还能再做两次。
**避免**：努力分、强度评分
```

## 规则

- **只在用户理解了之后才把术语加入。** 术语表是已被压缩的知识的记录，而不是用户用来学习的字典。如果用户刚被介绍到某个概念，等他们能够正确使用它之后，再把它收进来。
- **要有立场。** 当同一个概念存在多个用词时，挑出最好的那个，把其他的列为"应避免的别名"。**这就是语言压缩的方式。**
- **定义要紧凑。** 一两句话即可。定义这个术语 **是** 什么，而不是它做什么或如何去做。
- **在定义中使用术语表自己的术语。** 一旦某个术语进入了术语表，就在所有地方优先使用它——包括在其他术语的定义中。这能让复杂术语日后更易把握。
- **当出现自然的归并时，分小节归类**（例如 `## 解剖`、`## 训练编排`）。术语之间能自然贯通时，平铺一张表也可以。
- **明确标注歧义。** 如果某个词在更广的领域里被使用得很松散，注明本工作区的处理方式："在本工作区里，'set'（组）始终指一组工作组，热身组单独追踪。"
- **随着理解加深而修订。** 用户在第一周写下的定义到第六周可能就是错的。**就地更新；不要留下过期条目。**
