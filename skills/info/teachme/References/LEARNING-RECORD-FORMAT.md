# 学习记录格式

学习记录位于 `./learning-records/` 中，使用顺序编号：`01-slug.md`、`02-slug.md`，依此类推。该目录采用懒创建——只在写入第一条记录时才创建。

它们是教学版的 ADR（架构决策记录）：捕捉非显然的领悟、关键洞见，以及用户已声明的先验知识，用以指引未来的会话。它们用于计算最近发展区（ZPD）。

## 模板

```md
# {学到的或确立的内容的简短标题}

{1–3 句话：学到了什么（或确立了哪一项先验知识），以及它为何对未来的会话重要。}
```

这就是全部格式。一条学习记录可以只是一段话。它的价值在于记录：**这件事** 现在已知，以及 **为什么** 它会改变接下来该教什么——而不是把每个小节都填满。

## 可选小节

仅在它们真正带来价值时才加入。大多数记录都不需要这些。

- **Status**（前言区元数据）：`active | superseded by LR-NN`——在早期理解被证明错误并被替代时有用。
- **Evidence**：用户是如何展示出这种理解的（回答了某个问题、完成了某项练习、引用了过往经验）。当该主张可能被重新审视时记录这一项。
- **Implications**：这一记录为未来的会话解锁了什么、排除了什么。当含义不显然时值得记录。

## 编号

扫描 `./learning-records/` 找出现有的最大编号，加一即可。

## 何时写一条学习记录

满足以下任一条件时写一条：

1. **用户对某个非琐碎的内容展示了真正的理解**——不只是"接触过"，而是有证据表明他们能正确运用该概念。这为"接下来教什么"设定了一个新的下限。
2. **用户披露了先验知识**——"我已经会 X"。把它记录下来，使未来的会话不再重教。同时记录他们所宣称的**深度**。
3. **一个误解被纠正**——用户先前相信某个错误的东西，现在明白错在哪里。这类记录价值很高：它们能预测相关主题中未来可能出现的绊脚石。
4. **使命因学习而发生了转移**——用户发现自己关心的其实是另一件事。交叉链接到 mission.md 并更新它。

### 不构成学习记录的情形

- 仅仅是被覆盖到的材料。**覆盖不等于学习**。要等到出现证据再说。
- 已经在 glossary.md 中作为术语定义被简洁记录的内容。不要重复。
- 逐次会话的活动流水。学习记录不是日记——它们是足以驱动决策级别的洞见。

## 替代

当后一条记录与前一条相矛盾时（用户的理解加深了或被纠正了），把旧记录标记为 `Status: superseded by LR-NN`，而不是删除它。**理解如何演变的历史本身就是有用的信号。**
